# 원고 작업 공통 가이드

이 저장소는 단일 원고가 아니라, 여러 종류의 짧은 글·기고문·노트를 반복해서 작성할 때 재사용할 수 있는 최소한의 도구와 관례를 정리하는 것을 목표로 한다.

## 파일 구성과 역할

- `draft.md`  
  - 현재 작업 중인 원고의 초안 파일.  
  - 항상 **한 편의 글만**을 대상으로 하며, 작업이 끝난 글은 별도 저장소나 보관 위치로 옮기고 이 저장소에서는 제거한다.

- `outputs/`  
  - `draft.md` 등 원고를 pandoc으로 변환한 HTML·DOCX 등의 결과물이 저장되는 디렉터리.  
  - 언제든지 다시 생성할 수 있는 산출물이므로, 필요하면 통째로 삭제해도 된다.

- `references/`  
  - 원고 작성을 위해 참고하는 각종 보고서, 메모, 통계 등을 저장하는 폴더.  
  - 이 저장소에서는 **버전 관리 대상에서 제외**하는 것을 원칙으로 하고, 필요 시 별도 프라이빗 저장소나 클라우드에 보관한다.

- `count_words.py`  
  - 원고 분량(문자 수, 단어 수)을 간단히 측정하는 유틸리티 스크립트.

- `TODO.md`  
  - 저장소 자체(도구, 스크립트, 브랜치 전략 등)에 대한 할 일 목록을 기록하는 파일.  
  - 특정 원고에만 해당되는 일정·요청사항은 여기에 두지 않고, 해당 원고 전용 문서나 별도 저장소에 둔다.

## 원고 작업 흐름(예시)

1. **초기 세팅**
   - 새 원고를 시작할 때는 `draft.md`를 간단한 템플릿 수준으로 초기화한다.
   - 해당 원고에만 필요한 참고자료는 `references/` 폴더에 두되, 백업은 별도 위치에 해 둔다.

2. **집필과 분량 관리**
   - 집필은 주로 `draft.md`에서 진행한다.
   - 분량이 궁금할 때는 아래와 같이 실행한다.
     - `python count_words.py draft.md`

3. **출판용 포맷으로 변환**
   - pandoc을 이용하여 `draft.md`를 HTML, DOCX 등으로 변환하는 스크립트를 사용한다.  
   - HTML은 웹/메일 확인용, DOCX 등은 편집부 전달용으로 활용할 수 있다.

4. **정리**
   - 특정 원고 작업이 끝나면:
     - 원고와 관련 자료(초안, 최종본, 참고자료)는 별도의 보관 위치로 옮긴 뒤,
     - 이 저장소에서는 해당 원고에 한정된 내용과 파일을 삭제하고, 공통적으로 재사용 가능한 도구와 템플릿만 유지한다.

## 분량 관리 가이드(공통)

- **측정 도구**: `count_words.py`
- **실행 방법**:
  - `python count_words.py <파일명>`
- **활용 방법 예시**:
  - 초고–수정–최종본 단계마다 분량을 측정해, 요청받은 분량(예: 공백 포함 5,000–6,000자)을 대략적으로 맞춰 간다.
  - 서론·본론·결론 사이의 분량 비율이 크게 치우치지 않았는지 점검하는 용도로도 활용한다.
